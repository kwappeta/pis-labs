# Анализ границ ответственности и транзакций

**Вариант:** №19 (Книжный трекер)

## 1. Транзакционные границы
Транзакция начинается в **Book Service** при получении запроса на сохранение отзыва и завершается после успешного обновления статуса книги и пересчета рейтинга в базе данных.

## 2. Таблица анализа операций

| Операция | Синхронная/Асинхронная | Откат при ошибке | Retry-стратегия | Идемпотентность |
| :--- | :--- | :--- | :--- | :--- |
| Сохранение отзыва в БД | Синхронная | Да (ROLLBACK) | Нет | Да (по unique ID пользователя и книги) |
| Смена статуса книги | Синхронная | Да (в той же транзакции) | Нет | Да |
| Пересчет рейтинга книги | Синхронная | Да | Нет | Да |
| Публикация события в Event Bus | Асинхронная | Нет (Best-effort) | 5 попыток (Exponential backoff) | Да (дедупликация по event_id) |

## 3. Обработка исключений
- **Сбой БД**: Если база недоступна, транзакция не стартует, пользователь получает ошибку 500. Частичных данных не остается.
- **Сбой Event Bus**: Если сервис уведомлений недоступен, отзыв все равно считается сохраненным, а событие помещается в очередь для повторной отправки позже.
